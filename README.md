# MapShare

**MapShare** — это веб-платформа для создания, интерактивной визуализации и структурирования дорожных карт обучения в виде древовидных диаграмм. Проект представляет собой монолитный Full-stack комплекс на базе Laravel, Inertia.js и Vue 3. 

Вся работа с графом происходит на интерактивном холсте в режиме реального времени, обеспечивая бесшовный пользовательский опыт без полной перезагрузки страниц браузера благодаря протоколу Inertia. Платформа отлично подходит для онлайн-обучения, менторства и структурирования баз знаний для ИТ-сообществ.

## Возможности

### Модуль галереи и навигации (Для пользователей)
1. Регистрация: Возможность создать аккаунт для новых гостей системы.
2. Авторизация: Безопасный вход для зарегистрированных пользователей.
3. Публичная галерея: Просмотр коллекции общедоступных роадмапов, созданных сообществом.
4. Поиск карт: Поиск нужной дорожной карты по текстовому названию в реальном времени.
5. Режим Read-only: Интерактивный просмотр холста карты, масштабирование и перемещение без возможности случайного изменения элементов.

### Конструктор роадмапов (Функционал автора)
1. Интерактивный холст: Создание новых узлов (нод) и удаление существующих со всеми их связями.
2. Drag-and-Drop: Изменение координат положения узлов обычным перетаскиванием мыши.
3. Управление связями: Создание и удаление направленных стрелок (ребер) между узлами графа.
4. Редактирование контента: Ввод названий тем и подробного описания внутри узла.
5. Markdown в сайдбаре: Поддержка Markdown-разметки для контента узла с возможностью моментального переключения на вкладку предпросмотра отрендеренного HTML.
6 Переключение режимов (View/Edit): Динамическое скрытие элементов управления и инпутов сайдбара для сторонних пользователей или в режиме просмотра.

### Роадмапы пользователя и трекинг прогресса
1. Мои роадмапы: Отображение структурированного списка карт, созданных самим пользователем.
2. Удаление своих карт: Возможность безвозвратно удалить собственную роадмапу.
3. Фиксация индивидуального прогресса: При клике на узел в режиме просмотра можно отметить чекбокс «Изучено».
4. Цветовая индикация: Визуальное кодирование состояний узлов цветом и зачеркивание текста для отслеживания прогресса.

### Модерация (Для администраторов)
1. Удаление чужих карт: Администратор может модерировать контент и безвозвратно удалять любые роадмапы пользователей в один клик прямо из интерфейса галереи

## Технологии

### Frontend
* Vue 3 (Composition API): Реактивный интерфейс платформы.
* Vue Flow: Интерактивный холст для динамического рендеринга и работы с графами.
* Inertia.js: Протокол для бесшовного SPA-опыта без создания отдельного REST API.
* Markdown Parser: Парсинг и рендеринг markdown-текста внутри содержимого нод.

### Backend
* Laravel (PHP 8.x+): Основной серверный каркас и бизнес-логика.
* Inertia Laravel Adapter: Связующее звено для передачи данных на фронтенд.

### База данных
* PostgreSQL: Центральная база данных для синхронизации прогресса и хранения сущностей.
* JSONB Format: Хранение координат, типов связей, направленных стрелок и названий тем в нативном формате `JSONB` для минимизации затрат на серверную архитектуру.

## Установка и запуск

### Предварительные требования
Убедитесь, что на вашем компьютере установлены:
* PHP 8.1+ и менеджер пакетов Composer
* Node.js (16+) и npm
* СУБД PostgreSQL

### 1. Клонирование репозитория
```bash
git clone https://github.com/Ksorarenn/mapshare.git
cd franchaiserWeb
```
### 2. Настройка бэкенда (Laravel)
Установите необходимые PHP-зависимости:
```bash 
composer install
```
Создайте конфигурационный файл .env из шаблона:
```bash
cp .env.example .env
```
Сгенерируйте ключ приложения:
```bash
php artisan key:generate
```
### 3. Настройка базы данных
Создайте базу данных в PostgreSQL (например, mapshare_db). Откройте файл .env и пропишите параметры подключения к базе:
```sql
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=mapshare_db
DB_USERNAME=your_postgres_user
DB_PASSWORD=your_postgres_password
```
Запустите миграции и сидзеры для автоматического создания всех таблиц (users, roadmaps, node_contents, progress):
```bash
php artisan migrate --seed
```
### 4. Настройка фронтенда
Установите зависимости Node.js:
```bash
npm install
```
Запустите компиляцию фронтенда в режиме разработки для отслеживания изменений на лету:
```bash
npm run dev
```
### 5. Запуск сервера
В отдельном окне терминала запустите локальный сервер Laravel:
```bash
php artisan serve
```
По умолчанию проект станет доступен по адресу `http://127.0.0.1:8000`.
## Надежность и производительность
1. Высокая скорость работы: Ограничение времени загрузки и отрисовки холста карты составляет не более 2 секунд для графа из 150 узлов.
2. Валидация данных: Полная проверка всех вводимых данных в формах редактирования на стороне фронтенда.
3. Уведомления: Пользователь получает всплывающее сообщение об успешном сохранении любых изменений карты.
4. Масштабируемость MVP: Система рассчитана на поддержку до 1 000 карт, до 5 000 пользователей, до 150 узлов в одной карте.

## Устранение неполадок
### Ошибка подключения к базе данных (PostgreSQL)
1. Убедитесь, что служба PostgreSQL запущена на твоем сервере.
2. Проверьте корректность логина, пароля и имени БД в файле .env.
### Не сохраняются изменения графа
1. Проверьте сетевое соединение с бэкендом.
2. Убедитесь, что структура передаваемых координат и связей соответствует нативному формату JSONB в базе данных.
### Интерфейс отображается некорректно
1. Проверьте, запущена ли команда npm run dev в терминале.
